What is promise?

Promises provide a clean and robust way to handle asynchronous operations.

States: Pending, Resolved, Rejected.

Methods: then, catch, finally.

Chaining: Promises can be chained for sequential operations.

Multiple Promises: Use Promise.all and Promise.race to handle multiple promises concurrently.

Key Characteristics of Promises

Pending: The initial state, neither fulfilled nor rejected.

Resolved: The operation completed successfully, and the promise has a value.

Rejected: The operation failed, and the promise has a reason for the failure.

Synchronous | Asynchronous

Synchronous: means tasks happen one after another. Each task must wait for the previous one to finish before it can start.

Asynchronous: means tasks can happen independently of each other. A task can start and then move on to another task without waiting for the previous one to finish.

Async & Await in JavaScript:

async and await: are used to handle asynchronous operations more easily in JavaScript. They make asynchronous code look and behave like synchronous code, making it easier to read and write.
